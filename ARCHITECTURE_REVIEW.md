# Architecture Review
Generated: 2026-07-23T17:53:16.442Z
Score: 72/100

## Summary
The codebase follows a clear pages/components/lib/routes layering with consistent Pre-built-component conventions and no circular dependencies detected. However, several repeated defensive patterns (SDK-guard try/catch, haptic firing, localStorage CRUD boilerplate) were copy-pasted instead of shared, error-handling conventions differ between lib/api.ts and lib/storage.ts, and a few dead/orphaned files (unused shared/index.ts barrel, debug __tmp_check* files, likely-unused utils) linger in the tree.

## Issues
### Warnings
- [duplication] The 'SDK call may throw outside the App-in-Toss WebView, so wrap in try/catch and silently no-op/fallback' pattern is reimplemented independently in at least six places instead of being extracted to a shared helper. — Files: src/components/AdSlot.tsx, src/components/TossPurchase.tsx, src/components/TossRewardAd.tsx, src/components/BottomCTA.tsx, src/components/FloatingTabBar.tsx, src/lib/promo.ts / Fix: Introduce a shared `safeSdkCall(fn, fallback?)` utility in src/lib (e.g. src/lib/sdk.ts) that centralizes the try/catch-and-ignore behavior, and have each component/lib call it instead of hand-rolling try/catch blocks.
- [duplication] `fireHapticTickWeak` is defined identically (verbatim) in two page files, and BottomCTA.tsx has its own unexported `fireHaptic` with the same body — three separate implementations of the same one-liner haptic wrapper. — Files: src/pages/DiagnoseQuiz.tsx, src/pages/Problems.tsx, src/components/BottomCTA.tsx, src/components/FloatingTabBar.tsx / Fix: Create src/lib/haptics.ts exporting `fireHaptic(type: 'success' | 'tickWeak')`, export it from BottomCTA's needs too, and import it everywhere instead of redefining local copies.
- [duplication] lib/analytics.ts exports `computeRoi` (efficiency/progress/currentScore calculation) but Report.tsx does not use it — it reimplements the same efficiency and progress-ratio math inline with slightly different logic (e.g. using latestReal instead of latestExam-by-date). The two implementations can silently diverge, and computeRoi looks unused/dead. — Files: src/lib/analytics.ts, src/pages/Report.tsx / Fix: Have Report.tsx call computeRoi(sessions, exams, profile) for efficiency/progress instead of recomputing inline, or delete analytics.ts if it has been genuinely superseded by Report.tsx's logic.
- [consistency] Error-handling conventions differ across lib modules: lib/api.ts never throws and always normalizes failures to `{error: string}`; lib/storage.ts throws a raw Error for non-quota write failures but returns `{ok:false, reason:'quota'}` for quota failures specifically, while its read functions (getProfile/getMeta/etc.) swallow all errors and return null/defaults. Callers have to know which contract applies to which module. — Files: src/lib/api.ts, src/lib/storage.ts / Fix: Pick one convention for the whole lib layer (e.g. always return a discriminated Result type, never throw) and apply it uniformly to storage.ts and api.ts.
- [dead_code] src/shared/index.ts re-exports ScreenScaffold, useAppStore (renamed to useAppState), AppStoreProvider (renamed to AppStateProvider), and RouteState, but no other file in the codebase imports from '@/shared' — every page imports directly from '@/components/ScreenScaffold' and '@/lib/store'. This leaves an unused barrel that also introduces a confusing second name for the same hook/provider. — Files: src/shared/index.ts, src/lib/store.tsx / Fix: Delete src/shared/index.ts if it's unused, or commit to it as the single import surface and migrate all pages/components to import from '@/shared' instead of reaching into '@/lib/store' and '@/components/*' directly — don't keep both patterns and two names for the same symbol.
- [structure] src/__tmp_check.tsx and src/__tmp_check2.tsx are debug/scratch files sitting in the src root (not under pages/components/lib), unreferenced by App.tsx or routeTable.ts, and __tmp_check2.tsx unconditionally throws an Error — clearly leftover exploration code rather than shipped source. — Files: src/__tmp_check.tsx, src/__tmp_check2.tsx / Fix: Delete both files, or if they're intentionally kept for some tooling reason, move them out of src/ (e.g. into a scripts/ or scratch/ folder excluded from the build).

- [dead_code] `formatCurrency` and `cn` in src/lib/utils.ts are not referenced by any of the reviewed files — Amount.tsx and CountUp.tsx only use `formatNumber`. — Files: src/lib/utils.ts / Fix: Grep the full repo for `formatCurrency(` and `cn(` usage; remove either export if genuinely unused, or inline them at their call site if only used in one place.
- [naming] The double-underscore filename prefix is overloaded for two unrelated meanings: `__TdsGallery.tsx` marks an intentionally dev-only, env-gated route living inside the real src/pages directory, while `__tmp_check*.tsx` uses the same convention to mean 'scratch file that should probably be deleted'. This makes the underscore prefix ambiguous as a signal. — Files: src/pages/__TdsGallery.tsx, src/__tmp_check.tsx, src/__tmp_check2.tsx / Fix: Move dev-only tooling screens like TdsGallery to a dedicated src/dev/ folder outside src/pages, and remove ad-hoc scratch files entirely rather than prefixing them with underscores.
